PortfolioPage — глубина структуры

Нужны ли детальные страницы проектов или достаточно сетки со статусом?

A

Только сетка со статус-бейджами

Портфолио — это сетка карточек. На каждой: фото, название, площадь, теги, год и бейдж статуса (Реализован / В реализации / Концепт). Клик по карточке — ничего или якорь на brief.

Плюсы

  • Быстро в реализации
  • Нет контента для заполнения
  • Достаточно для MVP

Минусы

  • Нет детальной страницы проекта
  • Меньше доверия и вовлечённости
B

Сетка + детальная страница проекта

На карточке — клик → /portfolio/:slug. Детальная страница: галерея рендеров, площадь, что подбирали (мебель/свет/сантехника), статус, краткое описание задачи + решения.

Плюсы

  • Больше доверия через детали
  • SEO на каждый проект
  • Маршрут /portfolio/:slug уже есть

Минусы

  • Нужно заполнять контент проектов
  • Больше работы в реализации
