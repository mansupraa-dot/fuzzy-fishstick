# Research: Premium Furniture Website — Best Practices 2025–2026

> **Дата:** 2026-05-15
> **Скоуп:** Премиум / designer-сегмент. Hybrid-модель (каталог + лиды). Bilingual (RU + EN).
> **Цель документа:** База знаний для PRD. Не дизайн, не план реализации.
> **Что вне скоупа:** Конкретные технические решения (выбор CMS, headless-стека и т.п.), маркетинг-каналы, SEO-стратегия.

## A. Снимок решений проекта (2026-05-15)

> Это секция с конкретными ограничениями и решениями **именно нашего проекта**. Читать вместе с общим best-practices research ниже. Где-то общие паттерны нужно адаптировать под эти ограничения.

### A.1. Бизнес-модель и стратегия

- **Сейчас:** 90% выручки = дизайн-проекты, 10% = продажа мебели
- **Цель:** трансформация — перейти из дизайн-сервиса в **каталог-driven commerce**. Дизайн = текущий cash flow, который финансирует трансформацию.
- **Сайт = инструмент этой трансформации.** Не «лендинг для дизайн-услуги с каталогом», а **полноценное e-commerce ядро + lead-gen дизайн-сервис как поддержка**.
- **Целевой средний чек по каталогу:** 100–300k ₽
- **Команда:** один основатель — дизайнер/архитектор. Внешние дизайнеры на аутсорсе. Не штатные.

### A.2. Конкурентная позиция

| Конкурент | Что делает | В чём их слабость | Наше преимущество |
|---|---|---|---|
| [Aatom](https://aatom.ru/furniture/) | Премиум-производитель, 4 узких категории, нет цен, консультативная модель | Не e-commerce, узкий каталог, требует контакта менеджера | E-commerce с прозрачными ценами, шире каталог, кросс-категории |
| [Uniquely](https://uniquely.ru/) | Премиум-производитель, узкий куратор-каталог, цены есть, без фильтров | Узкая категоризация, нет фильтров, нет универсальности | Шире каталог, фильтры, более стильный визуал, кросс-категории |

**Наша уникальная ниша:** ни один из конкурентов не делает «universal home premium with full e-commerce». Aatom и Uniquely держат каталог узким, потому что они производители. **Мы — дистрибутор**, и это даёт стратегическое преимущество: можно продавать мебель + свет + сантехнику как «один источник для всего премиум-интерьера».

**Позиционирующая формула:** «Полный премиум-интерьер из одного места — мебель, свет, сантехника от лучших брендов. С возможностью получить дизайн-проект тут же».

### A.3. Контент и каталог

- **SKU:** 50 на старте → 300 через год
- **Источник товаров:** дистрибуция, не собственный бренд. Зависимость от фото поставщиков
- **Портфолио:** проекты в реализации + визуализации (рендеры). **Готовых проектов с pro-фото пока нет.** Это ограничение для §1.4.7 итальянской рекомендации «реальные интерьеры»
- **Журнал/блог:** редактора нет. **Не строим блог-инфраструктуру на старте** (см. §2.3 — мёртвый блог хуже, чем его отсутствие)
- **Pro фото/видео-съёмка каталога:** не готов пока. Используем supplier visuals. Это критичное ограничение для премиум-positioning — нужно компенсировать другими средствами (типографика, layout, портфолио).

### A.4. Сервис и логистика

- **1 шоурум** у партнёра
- **Сборка** — партнёрская (отдельная позиция в чекауте)
- **Срок доставки в регион** на карточке — **не реализуем** (нет ERP-интеграции). На карточке только общая информация о доставке.
- **Образцы материалов:** **не рассылаем**. На карточке указываем «образцы доступны в шоуруме» + кнопка «Записаться на визит»

### A.5. Стек

- **Архитектура:** Headless
- **Текущий React + Vite + Tailwind = MVP-этап**, не финальный стек. На production-этапе ожидается миграция на Next.js / Astro / Remix + headless CMS
- **AR / 3D / room planner** — фаза 2
- **CAD-интеграция (SketchUp/Archicad) для trade** — фаза 2
- **Trade Program формализованная** — фаза 2 (зависит от CAD-интеграции, которой нет)

### A.6. MVP scope (по ответам)

**В MVP:**
- ✅ Каталог: листинг + фильтры + карточка
- ✅ Корзина + checkout
- ✅ Личный кабинет
- ✅ Wishlist
- ✅ Brief-форма для лидов
- ✅ Карточка с упоминанием «образцы в шоуруме» + кнопка записи на визит (НЕ функция запроса образцов по почте)
- ✅ **2D-конфигуратор в карточке:** выбор ткани/цвета/размера со статичным превью (свотчи, доп.фото). НЕ 3D real-time preview.
- ✅ Базовое портфолио (рендеры + проекты в реализации)
- ✅ Базовая страница услуг / процесса / философии (для дизайн-трека)
- ✅ 1 шоурум (страница со временем работы, картой, записью на визит)

**Фаза 2:**
- 🔜 AR / 3D-просмотр
- 🔜 Реал-тайм 3D-конфигуратор
- 🔜 CAD-интеграция (SketchUp/Archicad экспорт)
- 🔜 Формализованная Trade Program
- 🔜 Журнал/блог (когда появится редактор)
- 🔜 Дополнительные шоурумы

**Не в скоупе пока:**
- ❌ Платная membership-программа (RH-style)
- ❌ Бесплатная рассылка образцов по почте
- ❌ Pro фото/видео-съёмка каталога
- ❌ Sourcebook-каталог в печатной форме

### A.7. Стратегические импликации для дизайна и PRD

1. **Hero главной должен балансировать два месседжа** — каталог премиум-интерьера + дизайн-услуга. Не выбирать что-то одно. Через год баланс сместится в сторону каталога, но сейчас 90/10 пока неизбежно.
2. **Каталог должен быть быстрым на конверсию** — это стратегическая ставка, на которую идём. Низкая friction, ясные цены, чёткие фильтры, кросс-селлинг между категориями (мебель → свет → сантехника).
3. **Brief-форма** — главный CTA в дизайн-треке, но не должен доминировать над «купить» в каталоге.
4. **Визуал — наша главная слабость.** Без pro-съёмки и без блога-журнала, выезжать придётся на **типографике + layout + curation + портфолио**. Дизайн сайта компенсирует слабость визуала каталога.
5. **«Universal home» — наш дифференциатор.** Кросс-категорийный мерчандайзинг (Hero «комплект для гостиной» включает диван + свет + декор) — это паттерн, на котором стоит строить главную и подборки.
6. **Trade program в фазе 2** — пока не педалируем, но архитектура должна это предусматривать (отдельный customer-type на уровне модели пользователя).

### A.8. Закрытые вопросы (2026-05-15)

- **E1. Целевая дата запуска MVP:** **Q3 2026**
- **A1. Имя бренда / домен:** **формируем в рамках проекта** (часть скоупа PRD/брендинга)
- **MVP-граница «портфолио»:** **включаем рендеры и визуализации** с честной атрибуцией. На странице портфолио каждый кейс должен быть помечен типом: «Реализован» / «В реализации» / «Концепт / Визуализация». Это критично для доверия — лучше открыто сказать «рендер», чем выдавать за фото.

## 0. Executive Summary

Что важно понять до того, как начнём писать PRD:

1. **Конверсия в премиум-сегменте низкая (1–2%) против 3–5% у mass-market.** Главное — не «продать в один клик», а провести через долгое исследовательское путешествие, дать достаточно контекста для уверенного решения. Покупатель сравнивает мебель неделями.
2. **Визуальный контент — главный конверсионный двигатель.** 79% участников исследований Baymard взаимодействуют с фото на странице товара ПЕРВЫМ делом (в обычном e-commerce — 46%). Lifestyle-снимки в интерьере побивают «студийку на белом» по любому показателю.
3. **AR/3D даёт измеримый рост:** +40% конверсии у тех, кто использовал AR (Wayfair: x3.4 к покупке), −60% возвратов, +25% AOV для конфигураторов. Но Baymard также показал, что 87% игнорируют «View in Room» — значит, AR не заменяет качественные фото, а дополняет.
4. **Hybrid-модель требует двух параллельных треков на сайте:** «купить готовое» (transactional) и «получить проект/консультацию» (lead-gen). Их нельзя сваливать в одну воронку.
5. **Russian premium → отдельный язык доверия.** «Образцы тканей», «тест-драйв», «в наличии в шоуруме», информация о сборке/доставке выше fold'а — то, что отличает успешные RU-кейсы (Hoff, Askona, Divan) от слабых.
6. **Storytelling — не «фишка», а фундаментальный слой.** Premium-бренды строят сайт как магазин + журнал + студия. Каталог без портфолио/процесса/философии в премиум-сегменте работает плохо.

#### 1.4.2. «Made in Italy» — юридический + маркетинговый щит

- **Закон Italy Law 135/2009** определяет «Made in Italy» как legal designation. Чтобы поставить лейбл, **весь процесс** (проектирование, производство, упаковка) должен происходить в Италии.
- Это значит: фраза «Made in Italy» на сайте — не маркетинг, а юридический trust-signal. Покупатели премиума это знают.
- Региональные специализации стоит называть прямо: «Made in Brianza», «Made in Tuscany» — это ещё сильнее, узнаваемо для премиум-аудитории.
- Использовать как наш паттерн: даже если мы дистрибутор, чётко атрибутировать страну/регион происхождения каждого товара. Не «европейская мебель», а «изготовлено в Тоскане, мастерская X».
- Цифры-якоря для storytelling: в топовых итальянских мастерских **до 70% работы делается ручными инструментами**, на сложное изделие уходит **120–180 часов**. Это не общие слова — это конкретные числа, которые продают.

#### 1.4.3. Inspiration-first vs hybrid: чему учиться у итальянцев

**У Minotti, Cassina, B&B Italia на сайтах нет цен и checkout.** Каталог = lookbook. Покупательский путь: вдохновился → нашёл sales network/showroom → пришёл в шоурум → договорился с дилером. Это **их выбор**, а не недоработка — они защищают premium-positioning через dealer-led модель.

Для нашего hybrid это означает:
- **Контент-страницы (Philosophy, Process, Portfolio, Journal, Collections)** — копировать итальянский editorial-tone полностью
- **Каталог + checkout** — оставлять как у нас (mid-range RU паттерн)
- **Карточка товара в hybrid** = вид-mash: верхняя половина в стиле Minotti (inspiration, проекты, designer story), нижняя половина — в стиле Hoff (размеры, доставка, варианты, купить)

Конкретные паттерны Minotti, которые стоит украсть:
1. **Navigation top-level: «Sales Network»** — не похоронен в footer. Для нас → видный пункт «Шоурумы» в header
2. **Products как «Focus On»** — кураторская подборка 4 модели с homepage, «Discover more» вместо «Add to cart»
3. **Projects сегментированы по сфере**: Residential / Hospitality / Hotel / Restaurant / Retail / Corporate / Yachting. Для нашего портфолио → жилые / коммерческие / общественные пространства / коммерческая недвижимость
4. **«Designers» как раздел top-level** — биографии, философия, связь с продуктами
5. **«Materials & Care»** — отдельный раздел про материалы и уход. Для премиума это часть инвестиционной логики «вещь надолго»
6. **Video Gallery** — отдельный раздел с производством, проектами, designer talks. Видео-контент → не «вспомогательный», а полноценный продуктовый класс

#### 1.4.4. Designer collaboration model

Итальянский паттерн: **дизайнер указывается всегда, на уровне продукта, коллекции и бренда.**
- Cassina: «Le Corbusier × Cassina», «Patricia Urquiola × Cassina»
- Molteni & C: единый креативный директор Vincent Van Duysen задаёт язык всему бренду
- Flexform: 45+ лет с Antonio Citterio как соавтор бренда
- Cappellini: brand = launchpad для дизайнеров (Morrison, Newson, Grcic стартовали здесь)

**Для нас:**
- Если работаем с дизайнерами интерьера / архитекторами — давать им страницу на сайте: имя, биография, проекты, работы. Это превращает их в со-авторов бренда.
- Если есть свой in-house дизайнер — он должен быть **лицом бренда**, не анонимным «мы»
- На карточке товара указывать дизайнера (если применимо) — даже для мебели массового производства

#### 1.4.5. Salone del Mobile — календарный якорь

- **Когда:** третья неделя апреля каждый год, Milan, Rho Fiera
- **Масштаб 2026:** 1 900+ брендов, 316 000 посетителей из 167 стран, 68% международные
- Это **главный мировой furniture-event**. Все премиум-новинки приурочены к нему.
- Для нашего контента: использовать Salone как событийный якорь даже без участия. «Salone 2026: что мы заметили», «5 трендов с Salone, которые мы привезли в Россию» — отличные пилларные публикации в журнал.
- Salone — то событие, по которому твой клиент-дизайнер интерьера ориентируется. Не игнорировать.

#### 1.4.6. Что НЕ копировать у итальянцев

- ❌ **Отсутствие цен и checkout** — для нашего hybrid это убийство одного из двух треков. У итальянцев это работает, потому что они продают через dealer network с десятилетиями репутации. У нас этого нет.
- ❌ **Только английский язык на основном сайте** — Cassina/Minotti делают это по своим причинам. Нам нужен RU как первичный.
- ❌ **Минимум информации на карточке** — итальянцы могут себе позволить «фото + название + дизайнер». Российский премиум-покупатель ждёт размеры, материалы, варианты, доставку.
- ❌ **Полный отказ от reviews/UGC** — итальянцы не показывают отзывы. Для нашего сегмента это слабость, а не сила: RU-покупатель хочет увидеть, что кто-то уже купил.

#### 1.4.7. Что взять напрямую (короткий чек-лист)

- [ ] Editorial-tone в storytelling-секциях (Philosophy, Process, Journal)
- [ ] Designer-страницы как top-level раздел
- [ ] «Sales Network» / «Showrooms» как видимый пункт навигации
- [ ] «Materials & Care» — отдельный информационный раздел
- [ ] Видео-галерея как продуктовый класс
- [ ] Архив/heritage-секция (для брендов с историей)
- [ ] Projects сегментированы по сферам (residential / hospitality / commercial)
- [ ] Designer credit на каждой коллекции/товаре
- [ ] Цифры-якоря для craftsmanship («120 часов на изделие», «N годов гарантии», «4 поколения мастеров»)
- [ ] Salone как контент-якорь в журнале

### 1.5. Американский premium — deep dive

Американский премиум-рынок устроен иначе, чем итальянский. Здесь нет дилерской монополии и закрытого инспирейшн-каталога — наоборот, e-commerce развит как нигде, и почти все ведущие бренды продают онлайн с ценами. При этом сегмент чётко делится на **три параллельные модели**, каждая с собственными правилами игры. Для нашего hybrid-проекта это самый ценный референс — здесь больше всего пересечений с тем, что мы строим.

#### 1.5.3. RH-модель — почему это другой жанр

RH — единственный кейс в мире, где furniture-ритейлер превратился в lifestyle-experience-бренд масштаба отеля. Это не «магазин», это destination. Ключевые элементы:

- **Sourcebook**: ежегодный печатный каталог 600+ страниц (в 2014 они выпустили 13 sourcebook'ов общим объёмом 2 500+ страниц). Это **lead-gen + brand artifact + журнал** одновременно. Запрос печатного sourcebook на сайте = capture email + физический контакт с брендом.
- **Galleries**: магазины переименованы в «галереи», часто в исторических зданиях, с интегрированными ресторанами, винными барами, отелями (RH Hotel в Newport Beach). Архитектура: итальянский мрамор, reclaimed wood, antique fixtures.
- **Website = digital magazine**: средняя сессия **12 минут против 4 минут** среднего furniture-ритейлера. Каждая карточка — long-form storytelling: история материала, craftsmanship, styling.
- **No public discounts**: премиум-positioning через отсутствие скидок. Скидки только через RH Members Program ($200/год → 25% от MSRP).
- **Members Program**: формализованная loyalty — не «бонусные баллы», а закрытый клуб.

**Что брать из RH для нашего hybrid:**
- ✅ Концепция annual sourcebook — PDF/printed lookbook привязанный к коллекциям. Запрос печатной версии = lead capture + физический touchpoint
- ✅ Editorial tone в карточке (long-form story, не bullets)
- ✅ Целевая метрика «time on site» как часть UX KPI (12 мин у RH достижимо именно через storytelling)
- ✅ Запись на визит в шоурум как премиум-experience, а не «магазин»

**Что НЕ брать:**
- ❌ Скрытие цен — у нас другая позиция в рынке, прозрачность важнее
- ❌ Membership за деньги — для запуска избыточно
- ❌ Гигантский каталог (десятки тысяч SKU) — для нашего размера это разрушит curated-positioning

#### 1.5.4. DTC craftsmanship — самая релевантная модель для нашего проекта

Maiden Home, Sixpenny, Interior Define — это новое поколение брендов 2014-2017 годов, которые **родились в e-commerce** и сразу строились как digital-first premium. Это **прямой референс для нашего hybrid**.

Их общие паттерны:

- **«Made in America» как trust signal**: Maiden Home — «handcrafted in North Carolina by artisans». Это не маркетинг, это origin story. Фото фабрики, мастеров, материалов.
- **Custom-to-order вместо склада**: Interior Define — 103+ ткани, конфигуратор в real-time. Заказ → 6-10 недель → доставка. Не «есть в наличии», а «сделают для вас».
- **No middleman narrative**: «We removed the showroom markup». Прямая цепочка фабрика → клиент → выигрыш в цене 30-50% против сравнимого retail.
- **Limited SKU**: Каждый бренд держит компактный каталог (Maiden Home — около 50 SKU, Burrow — около 30). Не «гипермаркет», а «галерея».
- **One-product-page-as-magazine**: каждая модель имеет очень длинную страницу с историей, материалами, видео мастеров, фотогалереей в интерьерах. Похоже на RH-pattern, но без масштаба галерей.
- **Free swatch program**: бесплатные образцы ткани приходят по почте за 5-7 дней. **Это premium-стандарт** в DTC-сегменте, не nice-to-have.
- **White-glove delivery + assembly**: дорого включено в цену, не отдельная позиция.

**Что брать для нашего hybrid:**
- ✅ «Made in [конкретный регион]» как чёткая атрибуция на карточке
- ✅ Free swatch / sample program (Divan уже делает это в RU — теперь усилить ссылками на американский паттерн)
- ✅ Real-time configurator с предпросмотром ткани/цвета (для своих коллекций)
- ✅ Compact curated catalog — не гнаться за объёмом SKU
- ✅ Long product pages с материалами, видео мастеров, реальными интерьерами
- ✅ White-glove delivery — отдельный value-prop, не строка в чекауте

#### 1.5.5. MillerKnoll и heritage-модель

В 2021 году Herman Miller купил Knoll → объединились в MillerKnoll. Это сейчас **крупнейший collective дизайнерских икон в мире**, владеющий Herman Miller, Knoll, DWR, HAY, Muuto, Maharam, HOLLY HUNT, NaughtOne и др.

Ключевые паттерны heritage-модели:

- **Дизайн-иконы как продуктовый класс**: Aeron Chair (1994), Eames Lounge (1956), Womb Chair (1948), Barcelona Chair (1929). Они продаются десятилетиями, цена $5-15k, спрос стабильный. На сайте они выделены в отдельный раздел «Iconic Designs» или «Classics».
- **Архивы как content engine**: MillerKnoll Archives — физическое пространство 12 000 sq ft в Michigan, **1 миллион объектов**. Архивные материалы → выставки → web content → social → возврат к продукту.
- **Designer biography as marketing**: Charles Eames, Ray Eames, George Nelson, Florence Knoll, Eero Saarinen, Mies van der Rohe — у каждого отдельная страница с биографией, философией, перечнем продуктов.
- **B2B + B2C через одну поверхность**: Herman Miller продаёт и Aeron в офис, и Eames Lounge домой. Каталог сегментирован по applications (Office / Home / Healthcare / Education), но визуальный язык единый.
- **«Better Together» (Knoll)** — целая контент-кампания о коллаборациях. Дизайнер + бренд = co-author.

**Что брать для нас:**
- ✅ Если у нас есть «икона» в каталоге (бестселлер, культовая модель) — выделять её визуально, давать ей отдельную карточку «коллекции»
- ✅ Биографии дизайнеров (если работаем с известными именами) — пересекается с итальянским паттерном (§1.4.4)
- ✅ Сегментация applications: «для дома / для офиса / для коммерческих пространств» — особенно если работаем с дизайнерами интерьеров жилых и коммерческих проектов

**Что НЕ брать:**
- ❌ Дублирующиеся «brand collective» страницы — у нас один бренд, не нужно эмулировать MillerKnoll-структуру
- ❌ Архивы на 100 лет — у молодого бренда это работает в обратную сторону

#### 1.5.6. Membership и Trade — формализация loyalty

Американский премиум впереди всех в формализации loyalty-программ:

| Программа | Кому | Условия |
|---|---|---|
| **RH Members Program** | Любому покупателю | $200/год → 25% от MSRP, ранний доступ к новым коллекциям |
| **CB2 Trade Program** | Дизайнеры/архитекторы | 20% от MSRP, без минималки, верификация по портфолио |
| **West Elm Trade** | Дизайнеры | 15-20% по объёму, доступ к B2B-каталогу, fabric library |
| **DWR Trade** | Дизайнеры | 15-20%, доступ к B2B-консьержу, технические файлы (CAD/SketchUp) |
| **Crate & Barrel Trade** | Дизайнеры/декораторы | До 20% по объёму, registry-функции для коммерческих проектов |

**Паттерны:**
- Скидка 15-25% — стандартный диапазон для trade
- Верификация через портфолио / лицензию / визитку — обязательна
- Доступ к **техническим файлам** (CAD, SketchUp, 2D dwg) — premium-маркер
- **Fabric/material library** — отдельный доступ к swatch-программе
- **B2B-консьерж** — выделенный менеджер для крупных клиентов
- Регистрация через форму, не через «звонок менеджеру»

**Для нас (§5.2 hybrid уже это упоминал, теперь конкретика):**
- Trade Program запускать в фазе 2, после стабильного B2C-потока
- Скидка 15-20% — реалистичный диапазон для российского рынка
- Технические файлы — критический value-prop для trade. Без них дизайнер не будет интегрировать наш каталог в свой SketchUp/Archicad
- Верификация — через ИНН + портфолио или ИП-статус

#### 1.5.7. Что НЕ копировать у американцев

- ❌ **Скрытые цены** (RH-style) — для российского hybrid это убийство конверсии
- ❌ **Massive sourcebook 600+ страниц** — у нас нет ресурсов на ежегодную полиграфию такого масштаба. Начинать с цифрового lookbook 30-50 страниц
- ❌ **Огромный SKU-каталог** (RH, Crate&Barrel — десятки тысяч) — у нас premium-curated, объём не наша сила
- ❌ **Membership за деньги на запуске** — нужна сначала база
- ❌ **«White-glove delivery» как включённый сервис** — в РФ логистика дороже, нужен отдельный селл
- ❌ **Длинные истории материалов на топовых позициях**, копирующие RH дословно — может звучать претенциозно в RU-локализации, нужна адаптация tone

#### 1.5.8. Что взять напрямую (короткий чек-лист)

- [ ] Sourcebook/lookbook как ежегодный/посезонный артефакт (PDF + опционально печатный для VIP)
- [ ] «Made in [регион]» атрибуция на карточке для каждого товара
- [ ] Free swatch / sample program — фото-видимая фича на карточке
- [ ] Real-time configurator (ткань/цвет/размер) для собственных коллекций
- [ ] Long-form product page с историей материала и видео мастеров
- [ ] White-glove delivery как отдельный value-prop
- [ ] Trade Program формализован: скидка 15-20%, верификация, CAD-файлы, fabric library, B2B-консьерж
- [ ] Сегментация applications в каталоге: для дома / коммерции / общественных пространств
- [ ] «Iconic Designs» как отдельная навигационная сущность (если применимо)
- [ ] Целевой KPI «time on site» — стремиться к 6+ мин (RH делает 12, mid-market делает 4)

## 2. Storytelling & контент-стратегия

### 2.1. Архитектура нарратива для премиум-бренда

Премиум-сайт работает на 4 слоях нарратива параллельно:

| Слой | Цель | Где живёт |
|---|---|---|
| **Philosophy / Manifesto** | Объяснить, почему мы существуем и во что верим | Главная (короткая версия), отдельная страница `/philosophy` или `/about` |
| **Process / Craft** | Показать, как делается работа — успокоить «как это собирается, кто гарантирует качество» | `/process`, `/how-we-work`, секция на карточке товара |
| **Portfolio / Case studies** | Соц.доказательство через готовые проекты, не «отзывы» | `/portfolio`, перелинковка с товарами |
| **Journal / Blog** | Поддерживать аудиторию между покупками, SEO, авторитет | `/journal`, `/blog` |

### 2.2. Что работает в премиум-storytelling (синтез источников)

**Эмоциональный visual narrative**
- Lifestyle-фото сильнее студии на белом фоне
- Видео и 360° создают «эффект живого ознакомления», повышают доверие (Askona)
- Photoshoot должен показывать «как живут с этой мебелью», не «как она стоит на складе»

**Behind-the-scenes аутентичность**
- Видео мастерской, материалов, людей-мастеров
- «Sourcing stories» — откуда дерево, ткань, фурнитура
- Sustainability / craft-этика (актуально для премиум 2025–26)

**Editorial structure для блога/журнала**
- 1 «pillar piece» в месяц (2 000+ слов, гайд по теме)
- Регулярный микро-контент (look of the week, designer interview)
- Длина адаптируется под платформу: коротко — соцсети, развёрнуто — сайт

**Авторство и дизайнерская биография**
- Карточка дизайнера у коллекции (DWR-паттерн)
- «Behind the design» — интервью с автором коллекции
- Это особенно важно для bilingual EN-аудитории, привыкшей к именованию авторов

**Practical + aspirational mix**
- Размеры, материалы, инструкции по сборке — рядом с inspiration-фото
- Не разделять каталог и журнал на параллельные миры — линкуйте взаимно

### 2.3. Что НЕ работает (антипаттерны)

- ❌ «Размытое преимущество»: «качественная мебель для вашего дома» — пустая фраза
- ❌ Stock-фото (любой premium-визитор отличает за 2 секунды)
- ❌ Бесконечная карусель «о нас», «миссия», «команда» без конкретики
- ❌ Блог-чулан со статьями раз в полгода и нерегулярными темами — лучше не иметь, чем иметь мёртвый
- ❌ Portfolio со скриншотами рендеров вместо реальных снимков готовых интерьеров

## 3. Catalog & Product Page UX

### 3.1. Каталог (листинг + фильтры)

**Faceted search — паттерны**
- Самые используемые фильтры — наверх (цена, бренд, для мебели — материал, габариты, цвет)
- Десктоп: persistent sidebar; мобайл: drawer modal, открывается по тапу
- Динамическое обновление без перезагрузки (Hoff делает это хорошо)
- Чекбоксы + multi-select, не radio
- Цена — slider + ручной ввод диапазона
- Длинные списки фильтров: показать 5–10, кнопка «Показать все», поиск внутри фасета
- Активные фильтры всегда видны (хлебные крошки фильтров)
- Скрывать пустые фасеты — не показывать варианты «0 товаров»

**Карточка в листинге**
- ≥3 фото по hover (Baymard: пользователи скроллят миниатюры до клика)
- Цена + старая цена + размер скидки
- Доступные варианты цвета/ткани мини-свотчами в карточке (без перехода в карточку — Hoff)
- Иконка «в наличии», «под заказ», «в шоуруме»
- Wishlist / «save for later» (премиум-сегмент решает долго, надо сохранять выбор)

**Сортировка и навигация**
- Стандартные опции: популярные, цена ↑/↓, новинки
- НЕ скрывать «всего товаров: N» — премиум-каталог обычно компактный, это плюс
- Категории через breadcrumbs всегда видимы
- «Готовые комплекты» / лукбук-подборки как отдельная навигационная сущность

### 3.2. Карточка товара — anatomy (синтез Cylindo + Baymard)

**Hero / визуальный блок** (главное)
- Многоракурсная галерея: ≥6 фото
- Обязательные типы: lifestyle (в интерьере), studio (на фоне), detail (фактура, фурнитура), dimensions overlay (с размерами), in-scale (рядом с человеком/объектом для масштаба)
- Zoom 1x→2x→4x (70% users считают zoom критичным)
- 360° — взаимодействие в 4 раза дольше статики, +6–8% конверсии
- Video walk-around — желателен для high-AOV
- Мобайл: те же возможности, что десктоп — никакого «упрощённого» мобильного

**Конфигуратор (если применимо)**
- Real-time preview: ткань, цвет, размер
- 71% готовы платить премию за кастомизацию
- Превью результата сохраняется в корзине и в email
- Если 3D — fallback на 2D-композиции цвета/материала

**Информационный блок**
- Заголовок, цена, варианты сразу под/рядом с фото
- Размеры — главная информация после цены: ширина × глубина × высота, посадочное место, высота сиденья
- Материал — простыми словами: не «100% Polyester 320 г/м²», а «бархат, мягкий на ощупь, износостойкий, годится для семей с детьми»
- Care instructions, гарантия, страна производства, дизайнер
- Что входит в комплект (для модульной — критично)
- Возможность заказа образца ткани/материала (Divan — бесплатно)

**Логистика на карточке** (Baymard: ДО checkout'а)
- Срок доставки в конкретный регион
- Стоимость доставки
- Сборка: есть/нет/доплата
- Возврат: окно, условия
- «Найти в шоуруме» — список адресов, статус наличия

**Доказательства и социальное**
- Отзывы со звёздами + фото-отзывы (Divan делает идеально)
- UGC из Instagram с тегом бренда
- «3 человека сейчас смотрят» / «купили на этой неделе N» — допустимо в премиуме, но осторожно
- «Использовано в проектах» — линкуйте на портфолио, где этот товар фигурирует (премиум-приём)

**CTA**
- Главный: «Добавить в корзину» / «Купить»
- Вторичный: «Запросить консультацию» / «Заказать у дизайнера»
- В hybrid-модели вторичный CTA важен — это вход в lead-gen воронку

**Cross-sell / комплектация**
- «Идёт в комплекте с» — другие товары из той же коллекции
- «Похожее в этой ценовой категории»
- «Завершите интерьер» — сопутствующие категории
- Recommendations повышают вероятность завершения покупки в 4.5x

### 3.3. Корзина и checkout

- 69% — средний показатель брошенных корзин, главный фактор — сложность checkout
- Возможность сохранить корзину (премиум-сегмент решает долго)
- Гость-чекаут обязателен (регистрация — опция, не блокер)
- Чёткие сроки доставки на финальной странице
- Множественные методы оплаты (для RU — карта, СБП, рассрочка)
- Промокоды и подарочные карты — поле, но не агрессивное

## 4. Дизайн-тренды 2025–2026

### 4.1. Визуальный язык

**Motion identity** — живой бренд
- Микро-анимации логотипа, заголовков, переходов
- Scroll-driven animation на ключевых секциях
- НЕ декоративно: каждое движение должно усиливать иерархию

**3D и depth**
- Интерактивные 3D-объекты в hero
- Объекты, реагирующие на scroll/hover
- Глубина через слои, тени, паралакс
- Cinema 4D / WebGL — для премиум-сегмента не overkill

**Editorial layouts**
- Свободные сетки, «разбросанные» элементы
- Большие fullbleed-фото
- Магазинная типография рядом с журнальной
- Воздух (white space) — премиум-маркер

**Типографика**
- Контрастные пары: humanist serif (заголовки, журнальное) + neutral sans (UI, продуктовое)
- Очень крупные размеры на hero и section openers
- Variable fonts для свободы

**Цвет и материал-палитры**
- Quiet luxury palette: терракот, охра, мшистый зелёный, нюдовый
- Earth tones доминируют над «modern minimalism» 2018–22
- Текстуры в фоны (бумага, штукатурка, лён)
- Чёрные тёмные секции для контраста с галереями фото

### 4.2. Интерактивные паттерны

- AR-просмотр («View in your room»)
- 3D-конфигуратор материалов/размеров
- Room planner (Askona — planner5D)
- «Снимок в интерьере» — пользовательская загрузка фото комнаты + наложение объекта (AI)
- Saved/wishlist с возможностью поделиться с дизайнером

### 4.3. Бренд-структура страницы

Современный премиум-главная экран чаще всего состоит из:

1. **Hero** — крупное визуальное обещание + 1 главное CTA + (опц.) категории
2. **Featured collection / new arrivals** — лимит 3–6 товаров
3. **Brand story teaser** — 1 секция с переходом на /philosophy

8. **CTA для лидов** — «Закажите консультацию» / «Расскажите о вашем проекте»

## 5. Hybrid-модель: каталог + лиды

### 5.1. Два параллельных трека на сайте

| Транзакционный трек | Lead-gen трек |
|---|---|
| Каталог → карточка → корзина → checkout | Услуги → бриф / квиз → консультация |
| Метрика: конверсия в покупку, AOV | Метрика: квалифицированные лиды, CPL |
| CTA: «Купить», «В корзину» | CTA: «Обсудить проект», «Получить расчёт» |
| Tone: продуктовый, конкретный | Tone: партнёрский, экспертный |

Эти треки **должны быть переплетены, но не смешаны**. На карточке товара — оба CTA (главный — «купить», вторичный — «обсудить проект»). На странице услуг — каталог не выпрыгивает, но возможен exit-link в каталог.

### 5.2. Lead-gen UX-паттерны

**Brief / quiz форма**
- 3–7 шагов с прогрессом
- Поэтапно: тип проекта → бюджет → стиль → сроки → контакт
- Без обязательного email до последнего шага (снижает дроп)
- Картинки вместо текста в выборе стиля
- Сохранение прогресса (cookie)

**Designer trade program** (B2B-трек внутри hybrid)
- Отдельная страница `/trade` или `/professionals`
- Регистрация дизайнеров с верификацией (загрузка визитки/портфолио)
- Скидка 15–20% (CB2 — 20%, без минималки)
- Доступ к 2D/3D файлам товаров, swatch-семплам, спецификациям (BoConcept-паттерн)
- Concierge-менеджер
- Pricing API для интеграции в Sketchup / AutoCAD
- Tiered loyalty: золото/серебро/бронза по обороту (Furnicher)

**Showroom integration**
- Адреса + карта на видном месте
- «Записаться на визит» с выбором времени
- «В наличии в шоуруме» индикатор на карточке (Askona)
- Виртуальный тур по шоуруму (если есть)

**Концепция «концьерж-сервис»**
- Кнопка «Помощь стилиста» в фиксированной плашке
- Чат с консультантом / WhatsApp
- Видео-консультация по записи
- «Закажите образцы материалов» (Divan — бесплатно)

### 5.3. Контент-воронка

```
Журнал/статья → Поджигает интерес
        ↓
Портфолио → Доказывает компетенцию
        ↓
Услуги/процесс → Объясняет, как работаем
        ↓
Бриф / контакт → Конвертирует
```

Параллельно: каталог → карточка → корзина (для готовых решений).

## 6. Trust signals и снятие риска

Мебель — high-AOV high-anxiety категория. Покупатель «не пощупал» товар. Топ-факторы успокоения:

### 6.1. Гарантии и пробный период
- Гарантия 2+ лет — стандарт; премиум-сегмент часто 5–10 лет; King Living — 25 лет
- Тест-драйв (Askona на матрасы) — снижает страх возврата

### 6.2. Отзывы и доказательства
- Verified reviews, желательно после 3–6 месяцев использования (Koala-паттерн)
- Фото-отзывы (Divan)
- UGC из соцсетей с упоминанием бренда (Divan)
- Кейсы из портфолио линкуйте на товары и обратно

### 6.3. Логистика и сборка
- Чёткие сроки и стоимость доставки на карточке, не на checkout
- Опция сборки (Askona — отдельная позиция, утилизация старой мебели)
- Тарификация по этажам/лифту/занос — premium-маркер
- Возвраты: окно ≥30 дней, чёткие условия

### 6.4. Шоурум как trust-anchor
- Адрес+карта вверху сайта (footer недостаточно)
- «Найти в шоуруме» на карточке
- Запись на визит
- Видео шоурума или 360°-тур

### 6.5. Прозрачность
- Реальные имена сотрудников/мастеров
- Реальные адреса производств
- Без анонимного «Произведено в Европе»

## 7. Технические соображения

### 7.1. Performance (Core Web Vitals)

Премиум-сайт = много тяжёлых фото и часто 3D. Это конфликт с CWV. План:

- **LCP < 2.5s** — критично. Каждые +1s сверху = −20% конверсии. Hero-изображение должно prelo­ad'иться
- **CLS < 0.1** — у всех `<img>`, `<video>` обязательно `width`/`height` или `aspect-ratio`
- **INP < 200ms** — заменил FID, важен для interactive 3D/конфигураторов
- Lazy load для всего below-the-fold
- Modern formats: AVIF / WebP, srcset для responsive
- Code splitting (Vite это даёт из коробки)
- 3D-модели должны загружаться по требованию (на клик по «3D View»), не на load

### 7.2. Bilingual (RU + EN)

- Не «копия страницы», а отдельный контент-stream
- Routing: `/ru/...`, `/en/...` (Next.js i18n / react-i18next)
- Hreflang теги для SEO
- Локализация валют, размеров (см vs inches), форматов адресов
- Разные офферы по регионам (доставка, шоурумы)
- Контент в журнале — частично разный (EN-аудитория ждёт фокус на дизайнерах, RU — на проектах)
- Картинки могут совпадать, но alt-теги разные
- Переключатель языка — видный, но не главный элемент header'а

### 7.3. Mobile-first и адаптив

- 60–70% трафика на мебельных сайтах идёт с мобилы
- Все 3D/AR функции должны работать на мобиле
- Touch-friendly: zoom, swipe, поворот 360°
- Мобильное меню (hamburger или нижняя полка)
- Sticky add-to-cart на карточке
- Filter drawer (а не sidebar)
- Не упрощать функционал на мобиле, делать ту же глубину

### 7.4. SEO для премиума

- Структурированные данные (Product schema, Review, Breadcrumb)
- OG-теги для шаринга в Pinterest/IG
- Pillar pages по категориям с длинным контентом
- Локальное SEO для шоурумов (LocalBusiness schema)
- Журнал = SEO-актив, не «секция», нужны категории, теги, поиск
- Скорость = SEO (CWV в ранжировании)

## 9. Вопросы и ответы для PRD

> Эти вопросы были собраны как open questions для разговора с заказчиком. Ответы зафиксированы 2026-05-15. Решения по этим ответам сведены в **секцию A** (наверху документа).

### 9.1. Бизнес и позиционирование
- **Кто 3 главных конкурента и наше отличие?** → См. A.2. Aatom + Uniquely. Наша ниша: universal home premium с full e-commerce + кросс-категории.
- **Целевой средний чек?** → 100–300k ₽
- **% выручки каталог vs. проекты, целевое соотношение?** → Сейчас 10/90. **Стратегическая цель: трансформация в каталог-driven.** Дизайн — текущий cash flow.
- **Свой штат дизайнеров или аутсорс?** → Один основатель-дизайнер + внешний аутсорс. Не штатные.

### 9.2. Контент и продукт
- **SKU на старте и через 12 месяцев?** → 50 → 300
- **Свои коллекции или дистрибуция?** → Дистрибуция (не собственный бренд)
- **Портфолио реализованных проектов?** → Проекты в реализации + визуализации (рендеры). Готовых pro-фото архивов нет.
- **Кто будет вести журнал?** → Никто пока. **Журнал не в MVP.**
- **Готовы инвестировать в pro-съёмку каталога?** → Нет пока. Используем supplier visuals.

### 9.3. Технологии
- **Headless или монолит?** → Headless
- **Текущий React + Vite — MVP-стек или прод?** → MVP. Прод-стек определяется отдельно.
- **AR/3D на запуске или фаза 2?** → Фаза 2
- **CAD-интеграция для trade program?** → Фаза 2

### 9.4. Логистика и сервис
- **«Срок доставки в регион» на карточке?** → Не реализуем
- **Сборка собственная или партнёрская?** → Партнёрская
- **Сколько шоурумов и где?** → 1 шоурум у партнёра
- **Бесплатная рассылка образцов?** → Нет. Образцы доступны в шоуруме.

### 9.5. Запуск и MVP scope
- **MVP-модули?** → Каталог + карточка + корзина + checkout + Личный кабинет + Wishlist + Brief-форма + упоминание «образцы в шоуруме» + 2D-конфигуратор (выбор цвета/ткани/размера со статичным превью) + базовые страницы (Услуги, Процесс, Философия, Шоурум, Портфолио из рендеров)
- **Фаза 2:** AR/3D, реал-тайм 3D-конфигуратор, CAD-интеграция, Trade Program, Журнал
- **Целевая дата запуска?** → **Q3 2026**
- **Имя бренда / домен?** → Формируем в рамках проекта (часть брендинг-скоупа)
- **Портфолио в MVP — реальные проекты или рендеры?** → Включаем рендеры/визуализации с честной атрибуцией: «Реализован» / «В реализации» / «Концепт»
