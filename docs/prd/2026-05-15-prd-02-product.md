## §1. User Journeys

### Journey 1 — Homeowner (transactional track)

```
Discovery
  └─ Google / Pinterest / соцсети → Home или Product page напрямую

Исследование
  └─ Каталог → фильтры (категория, цена, бренд, материал)
  └─ Карточка → фото галерея → 2D-конфигуратор (цвет/ткань/размер)
  └─ Wishlist ← «сохраню, подумаю»

Доверие
  └─ Портфолио — смотрит реализованные проекты
  └─ «Образцы в шоуруме» → запись на визит (необязательно)
  └─ Отзывы на карточке (фаза 2 — пока trust через бренды)

Решение
  └─ Корзина → Checkout
  └─ Оплата: Карта или СБП
  └─ Подтверждение + email

Exit points:
  └─ Wishlist (отложил, вернётся позже)
  └─ Бриф-форма (решил взять дизайнера)
```

---

### Journey 2 — Дизайнер/Архитектор (lead-gen track)

```
Discovery
  └─ Личная рекомендация / портфолио проекта → Home или Portfolio

Исследование каталога
  └─ Каталог → несколько товаров → Wishlist (подборка для клиента)
  └─ Карточка → технические характеристики, габариты

Контакт
  └─ Страница Услуг → Бриф-форма
  └─ Или: карточка → «Обсудить проект» → Бриф-форма

Конверсия
  └─ Заполненная бриф-форма → менеджер (основательница) связывается
  └─ Ручной процесс в MVP: email / WhatsApp

Exit points:
  └─ Покупка напрямую через корзину (без бриф-формы)
  └─ Wishlist без оформления (вернётся при следующем проекте)
```

---

### Journey 3 — Gift Buyer (быстрый transactional)

```
Discovery
  └─ Соцсети / прямая ссылка → Product page

Решение (быстрое)
  └─ Фото → цена → «Купить»
  └─ Корзина → Checkout (guest, без регистрации)
  └─ Оплата → Email-подтверждение

Барьеры, которые нужно убрать:
  └─ Обязательная регистрация перед покупкой ❌
  └─ Запрос лишних данных в форме ❌
  └─ Непонятные условия доставки ❌
```

## §3. Page Specs

### 3.1 Home (Главная)

**Цель страницы:** Первое впечатление — транслирует «универсальный премиум-интерьер из одного места». Балансирует два трека: каталог (primary) и дизайн-услуга (secondary).

**Секции (сверху вниз):**

| # | Секция | Описание | CTA |
|---|---|---|---|
| 1 | **Hero** | Fullscreen. Lifestyle-фото интерьера (гостиная/спальня). Заголовок + подзаголовок по позиционирующей формуле. | «Смотреть каталог» (primary) + «Обсудить проект» (secondary) |
| 2 | **Featured Collection** | 3–4 выбранных товара. Мегаменю-превью трёх категорий. | «В каталог» |
| 3 | **Cross-category strip** | «Мебель + свет + сантехника» — кросс-категорийный месседж. Иконки или фото трёх категорий. | Ссылки на разделы |
| 4 | **Portfolio teaser** | 2–3 проекта из портфолио. Каждый с атрибуцией. | «Все проекты» |
| 5 | **Philosophy teaser** | 2–3 предложения бренд-манифеста + фото основательницы или детали интерьера. | «О нас» |
| 6 | **Services / Process** | Короткое объяснение дизайн-трека. 3 иконки-шага. | «Рассказать о проекте» → Brief-форма |
| 7 | **Trust strip** | Иконки: гарантия / доставка / шоурум / N брендов. | «Шоурум» |
| 8 | **Collections teaser** | 2 кураторские подборки. | «Все коллекции» |

**Примечания:**
- Блок журнала/статей — отсутствует (нет редактора)
- Hero фото — из портфолио (рендер с атрибуцией «Реализован / Концепт»)

---

### 3.2 Catalog (Листинг)

**Цель:** Помочь найти нужный товар с минимальным числом шагов.

**Layout:**
- Десктоп: sidebar фильтры (слева, фиксированные) + сетка 3–4 колонки
- Мобайл: drawer фильтры (по кнопке) + сетка 2 колонки

**Фильтры (приоритет):**
1. Категория (если не выбрана)
2. Цена (слайдер + ручной ввод)
3. Бренд (чекбоксы, поиск внутри)
4. Материал (мебель: ткань/кожа/дерево/металл)
5. Цвет (свотчи)
6. В наличии / Под заказ
7. Размер (габариты для мебели)

**Карточка в листинге:**
- Фото (hover → второе фото)
- Бренд + название
- Цена
- Мини-свотчи доступных вариантов
- Иконка wishlist
- Бейдж: «В наличии» / «Под заказ» / «В шоуруме»

**Сортировка:** Популярные / Цена ↑ / Цена ↓ / Новинки

**Счётчик:** «Найдено: N товаров» — всегда виден.

---

### 3.3 Product Page (Карточка товара)

**Цель:** Дать достаточно информации для уверенной покупки без визита в шоурум.

**Layout:** Двухколонный выше fold (галерея слева, инфо справа), затем полная ширина.

**Блоки:**

**① Галерея (левая колонка)**
- ≥ 5 фото: lifestyle в интерьере, studio, детали, масштаб рядом с человеком
- Zoom при клике
- Мобайл: swipe-галерея

**② Информационный блок (правая колонка)**
- Бренд + название + дизайнер (если применимо)
- Цена
- **2D-конфигуратор:** свотчи ткани/цвета + выбор размера → статичный превью (другое фото, не real-time 3D)
- «Образцы доступны в шоуруме» + «Записаться на визит»
- **CTA primary:** «Добавить в корзину»
- **CTA secondary:** «Обсудить проект» → Brief-форма
- Wishlist-кнопка

**③ Характеристики (ниже fold)**
- Размеры: Ш × Г × В, высота сиденья, посадочная глубина
- Материал: описание простыми словами (не технические коды)
- Что входит в комплект
- Страна производства / регион
- Гарантия
- Уход и обслуживание

**④ Доставка и сервис**
- Способы доставки (без срока в конкретный регион — нет ERP)
- Сборка: партнёрская, отдельная позиция в корзине
- Возврат: условия

**⑤ «Использован в проектах»**
- 1–2 карточки из портфолио, где этот товар фигурирует
- Линк на полный проект

**⑥ Cross-sell**
- «Из той же коллекции» — товары того же бренда/серии
- «Завершите интерьер» — смежные категории (диван → торшер → ковёр)

---

### 3.4 Cart (Корзина)

**Требования:**
- Список товаров с фото, названием, вариантом, ценой, количеством
- Редактирование количества и удаление
- Сохранение корзины (cookie, без логина)
- Опция сборки — чекбокс «Добавить сборку» с ценой
- Итог: сумма + доставка (уточняется на следующем шаге)
- CTA: «Оформить заказ» (primary) + «Продолжить покупки»
- Wishlist-перенос: «Сохранить на потом»

---

### 3.5 Checkout

**Структура (одна страница с секциями или 3-step):**

```
Шаг 1: Контакт
  — Имя, телефон, email
  — Guest checkout (регистрация опциональна)

Шаг 2: Доставка
  — Адрес доставки
  — Выбор варианта доставки (курьер / самовывоз)
  — Сборка: да/нет

Шаг 3: Оплата
  — Карта (Visa / MC / Мир)
  — СБП
  — Итог заказа (резюме)
  — Кнопка «Оплатить»
```

**Правила:**
- Без обязательной регистрации до оплаты
- Прогресс-бар шагов всегда виден
- Поля минимальны: только необходимые данные
- После оплаты: страница «Спасибо» + email-подтверждение

---

### 3.6 Account + Wishlist

**Минимальный MVP-скоуп:**

| Функция | В MVP |
|---|---|
| Регистрация / логин | ✅ email + пароль |
| История заказов | ✅ |
| Wishlist (избранное) | ✅ сохранение + удаление |
| Поделиться wishlist | ✅ по ссылке (для дизайнеров) |
| Личные данные / адреса | ✅ |
| Trade-кабинет | ❌ фаза 2 |
| Программа лояльности | ❌ фаза 2 |

### 3.7 Brief-форма (Lead-gen)

**Цель:** Квалифицировать лид для дизайн-проекта.

**Структура (5 шагов):**

```
Шаг 1: Тип проекта
  — Квартира / Дом / Коммерческое пространство / Другое

Шаг 2: Площадь и помещение
  — Площадь: <50 / 50–100 / 100–200 / 200+ м²
  — Какие комнаты: гостиная / спальня / кухня / ванная / всё

Шаг 3: Бюджет
  — до 500k / 500k–1.5M / 1.5M–5M / 5M+ ₽

Шаг 4: Стиль
  — Визуальный выбор: 4–6 картинок-референсов (минимализм / сканди / классика / авангард / эклектика)

Шаг 5: Контакт
  — Имя, телефон, удобный способ связи (WhatsApp / звонок / email)
  — «Расскажите дополнительно» (опц. textarea)
```

**UX-правила:**
- Прогресс-бар (Шаг N из 5)
- Email не запрашиваем до последнего шага
- Сохранение прогресса в cookie
- После отправки: «Спасибо — свяжемся в течение 24 часов»

---

### 3.8 Portfolio

**Цель:** Соцдоказательство экспертизы. Работает как trust-якорь перед покупкой.

**Листинг:**
- Сетка проектов: превью-фото + название + тип + атрибуция
- Фильтр по типу: Жилые / Коммерческие / Концепты

**Карточка проекта:**
- Hero-фото / рендер
- **Атрибуция (обязательно):** бейдж «Реализован» / «В реализации» / «Концепт / Визуализация»
- Описание: площадь, локация (город), задача
- Галерея рендеров/фото
- Используемые товары из каталога (linkout на карточки)

**Правило честности:** Рендеры никогда не выдаются за фото. Атрибуция видна без дополнительного клика.

---

### 3.9 Showroom

**Цель:** Снизить тревогу перед крупной покупкой. Образцы в шоуруме = замена free-swatch-программы.

**Содержимое:**
- Адрес + карта (Яндекс.Карты embed)
- Часы работы
- Контактный телефон
- Фото шоурума (интерьер, образцы материалов)
- «Здесь можно потрогать образцы тканей и материалов»
- Форма записи на визит: имя + телефон + дата/время (простой выбор слота)

---

### 3.10 Services / Process / Philosophy

**Services (Услуги):**
- Два блока: «Каталог» (e-commerce) + «Дизайн-проект» (lead-gen)
- Для дизайн-трека: что входит, этапы, примерный диапазон стоимости (опц.)
- CTA: «Рассказать о проекте» → Brief-форма

**Process (Как мы работаем):**
- 4–5 шагов: Знакомство → Бриф → Концепция → Реализация → Результат
- Визуальная шкала / иконки
- Конкретные сроки и форматы работы

**Philosophy (О нас / Манифест):**
- Кто основательница: имя, специализация, подход
- Почему этот бизнес: личная история трансформации из дизайн-сервиса в каталог
- Принципы: кросс-категорийный курирование, прозрачные цены, честная атрибуция
- Фото основательницы

## §4. Что НЕ в MVP (явный список)

| Функция | Причина | Когда |
|---|---|---|
| AR / 3D-просмотр товара | Дорого, фаза 2 | Фаза 2 |
| Real-time 3D-конфигуратор | Зависит от AR/3D | Фаза 2 |
| Trade Program / B2B-кабинет | Требует CAD-интеграции | Фаза 2 |
| Блог / Журнал | Нет редактора | Фаза 2 |
| Рассрочка / BNPL | Не в приоритете MVP | Фаза 2 |
| EN-версия сайта | Нет аудитории пока | Фаза 2 |
| Отзывы пользователей | Нет накопленной базы | Фаза 2 |
| Дополнительные шоурумы | Нет партнёров | Фаза 2 |
| Поиск по фотографии | Технически сложно | Фаза 2+ |
| B2B-счёт / предоплата юрлица | Ручной процесс пока | Фаза 2 |
| UGC из соцсетей | Нет накопленного контента | Фаза 2 |
