**Статус:** MVP-плейсхолдер. Не переоптимизировать под прод-задачи — скоро миграция.

### Production-стек (планируется, выбор в фазе implementation)

| Слой | Варианты |
|---|---|
| Framework | Next.js 14+ / Astro / Remix |
| CMS (headless) | Sanity / Strapi / Contentful |
| Платежи (эквайринг) | ЮKassa / CloudPayments / Тинькофф |
| Поиск/фильтры | Algolia / MeiliSearch / встроенный |
| Email | Mailchimp / SendPulse / Postmark (транзакционный) |
| Аналитика | Яндекс.Метрика (обязательно) + GA4 (опц.) |
| CDN | Cloudflare / Vercel Edge |

### i18n

- **MVP:** только RU
- **Архитектура:** с первого дня строить с i18n-ready подходом (строки через i18n-ключи, не хардкод)
- **Фаза 2:** EN добавляется без переписывания

### 2D-конфигуратор (карточка товара)

- **Что это:** набор свотчей (ткань/цвет/материал) + выбор размера
- **Поведение:** при выборе варианта меняется фото в галерее (статичное, не 3D)
- **Реализация:** массив вариантов в данных товара, каждый вариант = свой набор фото
- **НЕ делаем:** real-time 3D-рендер, WebGL, AR-наложение

### Core Web Vitals (цели)

| Метрика | Цель |
|---|---|
| LCP | < 2.5s |
| CLS | < 0.1 |
| INP | < 200ms |

**Критично:** Hero-изображение должно preload'иться. Все `<img>` с явными `width`/`height`.

## §3. Операционные ограничения

### Один человек = операционный потолок

Сайт должен быть управляем одним человеком без найма операционной команды:

| Задача | Как решается в MVP |
|---|---|
| Добавление товаров | Через CMS (в production-стеке). В MVP — редактирование data-файлов |
| Обработка заказов | Email-уведомления + ручная обработка |
| Ответы на бриф-формы | Email-уведомление → личный ответ |
| Обновление портфолио | Через CMS. 1–2 проекта в квартал |
| Поддержка каталога | 50 SKU → не более 300. Рост контролируемый |

### Каталог: правила наполнения

- **Старт:** ≥ 30 SKU перед запуском волны 1 (меньше — незаполненный каталог убивает доверие)
- **Рост:** 50 SKU → 150 к концу Q3 → 300 к концу 2026
- **Фото:** supplier visuals. Минимум 3 фото на SKU (studio + lifestyle + деталь)
- **Данные товара:** обязательны — цена, габариты (Ш×Г×В), материал, страна, варианты

### Доставка и сборка

- Доставка: курьерская до двери. Партнёр TBD.
- Сборка: партнёрская, отдельная позиция в чекауте.
- **Срок доставки в конкретный регион:** не выводим на карточке (нет ERP). Только общая информация.
- Возврат: стандартные 14 дней по закону + условия для крупногабарита.

### Шоурум

- 1 шоурум у партнёра
- Образцы материалов — только там, не рассылаем
- Запись на визит — через форму на сайте, подтверждение вручную или простым calendar-виджетом

## §4. Риски и митигация

| Риск | Вероятность | Последствие | Митигация |
|---|---|---|---|
| **Визуал каталога слабый** (нет pro-фото) | Высокая | Низкое доверие к товарам | Компенсировать дизайном карточки + портфолио + честная атрибуция supplier-фото |
| **Мало SKU на старте** | Средняя | Каталог выглядит пустым | Не запускать волну 1 до ≥30 SKU. Лучше позже, но заполнено. |
| **Один исполнитель = узкое место** | Высокая | Задержка волн | Жёсткое соблюдение scope. Фаза 2 не проникает в MVP ни при каких условиях. |
| **Эквайринг не подключён к запуску** | Средняя | Checkout не работает | Подключить эквайринг до старта волны 1 — блокирующее требование |
| **Домен/бренд-имя не выбраны** | Средняя | Задержка всего запуска | Naming — следующий этап сразу после PRD. Блокирует дизайн логотипа. |
| **Фаза 2 scope проникает в MVP** | Средняя | Задержка запуска | Явный список «не в MVP» (§4 prd-02-product.md) как контракт |
| **Нет накопленных отзывов** | Высокая | Слабый trust-сигнал | Компенсируем брендами поставщиков, портфолио, честной коммуникацией |
