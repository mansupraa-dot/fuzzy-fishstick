## §1. Vision & Mission

**Vision (куда идём):**
Стать главным онлайн-пространством для комплектации премиум-интерьера в России — место, где покупатель получает мебель, освещение и сантехнику лучших брендов и при желании — дизайн-проект от того же источника.

**Mission (зачем сайт):**
Трансформировать бизнес из дизайн-сервиса (90% выручки) в каталог-driven commerce (цель — большинство выручки). Сайт — инструмент этой трансформации. Дизайн-услуга остаётся, но переходит в роль поддержки каталога, а не наоборот.

**Позиционирующая формула:**
> «Полный премиум-интерьер из одного места — мебель, свет, сантехника от лучших брендов. С возможностью получить дизайн-проект тут же.»

### Наша уникальная ниша

Ни один прямой конкурент не делает «universal home premium с полным e-commerce». Aatom и Uniquely держат каталог узким — они производители. **Мы дистрибутор**, и это стратегическое преимущество: можно продавать мебель + свет + сантехнику как единый источник для премиум-интерьера.

Референсы, на которые смотрим:
- **Контент-тон:** Minotti, Flexform (итальянский editorial)
- **E-commerce UX:** Maiden Home, Divan.ru (конфигуратор, фильтры, trust-сигналы)
- **Визуальное направление:** Light Spatial Glass (см. prd-03-brand.md)

## §3. Персоны

### Персона 1 — Homeowner «Ремонт/Переезд»

**Кто:** Частное лицо, 30–50 лет, обустраивает новую квартиру или дом. Принимает решения самостоятельно, иногда с партнёром. Бюджет на мебель 200–600k ₽.

**Цели:**
- Найти качественную мебель с ясными ценами без звонка менеджеру
- Увидеть, как товар выглядит в реальном интерьере
- Получить уверенность перед крупной покупкой

**Боли:**
- Страх купить онлайн дорогую вещь «вслепую»
- Сложность сравнения нескольких вещей из разных магазинов
- Неясные сроки и условия доставки

**Вход в воронку:** Google/соцсети → каталог → карточка → корзина → checkout

**Ключевые страницы:** Home, Catalog, Product page, Cart, Checkout, Wishlist

**Главный CTA:** «Добавить в корзину» / «Записаться на визит в шоурум»

---

### Персона 2 — Дизайнер интерьера / Архитектор

**Кто:** Профессионал, покупает для клиентского проекта. ИП или физлицо. Закупает 3–15 позиций на проект, средний проект 500k–3M ₽.

**Цели:**
- Быстро найти нужное в каталоге и добавить в вишлист для согласования с клиентом
- Получить технические данные (габариты, материалы) для вписки в проект
- Договориться об условиях работы (скидка, счёт, сроки)

**Боли:**
- Нет trade-скидок в MVP (фаза 2) — временный барьер
- Нужны точные спецификации, которые часто спрятаны
- Долгое согласование с клиентом без возможности показать подборку

**Вход в воронку:** Личная рекомендация / портфолио → каталог → wishlist → бриф-форма

**Ключевые страницы:** Catalog, Product page, Wishlist, Brief-форма, Portfolio

**Главный CTA:** «Сохранить в wishlist» / «Обсудить проект»

> **Примечание:** Trade Program (скидки, CAD-файлы, B2B-кабинет) — фаза 2. В MVP эта персона работает через стандартный каталог + ручной контакт через бриф-форму.

---

### Персона 3 — Gift / Occasion Buyer

**Кто:** Покупает один конкретный предмет — подарок, замена сломанного, импульс от красивой фотографии. Чек 30–150k ₽. Решение принимает быстро.

**Цели:**
- Купить конкретную вещь с минимальным числом шагов
- Убедиться в надёжности продавца (доставка, возврат)

**Боли:**
- Долгий checkout отпугивает
- Нет уверенности в качестве без возможности «потрогать»

**Вход в воронку:** Соцсети / Pinterest → карточка → быстрый checkout

**Ключевые страницы:** Product page, Cart, Checkout

**Главный CTA:** «Купить» (минимум шагов)

## §4. Бизнес-модель и ключевые ограничения

### Бизнес-модель

| Трек | Описание | Текущая доля | Целевая доля |
|---|---|---|---|
| **Каталог (transactional)** | Продажа мебели/света/сантехники онлайн | 10% выручки | Majority |
| **Дизайн-услуга (lead-gen)** | Дизайн-проект по брифу | 90% выручки | Cash flow в период трансформации |

Сайт строится как **e-commerce ядро** с дизайн-услугой как вторичным треком, а не наоборот.

### Жёсткие ограничения MVP

| Ограничение | Последствие для продукта |
|---|---|
| Один основатель (дизайнер/архитектор) | MVP операционно подъёмен одним человеком |
| Нет редактора блога | Блог/журнал не в MVP. Вместо него — кураторские Collections |
| Нет бюджета на pro-съёмку каталога | Используем supplier visuals. Компенсируем дизайном и портфолио |
| Дистрибутор, не производитель | Нет behind-the-scenes мастерской, нет собственных коллекций |
| 1 шоурум партнёра | Образцы только в шоуруме — не рассылаем по почте |
| Нет ERP-интеграции | Срок доставки в конкретный регион не выводим на карточке |
| Стек React+Vite = MVP, не прод | Не переоптимизировать текущий код под прод-задачи |

## §5. KPI успеха MVP

**Горизонт измерения:** 3 месяца после запуска (конец Q3 + первый месяц Q4 2026)

### Первичные KPI (must hit)

| KPI | Описание | Целевое значение |
|---|---|---|
| **Покупки через сайт** | Завершённые заказы через checkout | ≥ 3 заказа/мес |
| **Лиды на дизайн-проект** | Заполненные бриф-формы с контактом | ≥ 5 квалифицированных/мес |

### Вторичные KPI (track but don't optimize for yet)

| KPI | Описание |
|---|---|
| Время на сайте | Цель: ≥ 4 мин (RU mid-market норма), стремимся к 6+ |
| Глубина каталога | Среднее число страниц товаров за сессию |
| Конверсия бриф-формы | % посетителей страницы услуг → заполнили форму |
| Wishlist-активность | Дизайнеры сохраняют подборки → сигнал trade-спроса |
| Запись на визит в шоурум | Онлайн-записи через форму на сайте |

### Что НЕ является KPI MVP

- SEO-позиции (органика придёт позже)
- Число SKU в каталоге (качество > количество)
- Социальные подписчики
